╚═══════════════════════════════════════════════════════════════════════════════════════════════╝

1. LEFT COLUMN: Shows ALL users found (always visible)

2. RIGHT COLUMN: Shows folders for ONLY the selected user
   • Empty until you click a user
   • Updates when you click a different user
   • Shows ONLY one user's folders at a time

3. This is NOT a table where you see all users' folders simultaneously
   • You can only see one user's folder list at a time
   • Like macOS Finder columns or Windows Explorer navigation

4. WORKFLOW:
   Step 1: Scan → Left column populates with all users
   Step 2: Click alice → Right column shows alice's 4 folders
   Step 3: Click bob → Right column REPLACES with bob's 1 folder
   Step 4: Select folders in right column → Enable remove button

RIGHT PANE BECOMES TABBED:

  Tab 1: "Search Access" (NEW)
    ├─ Search form (email, keyword, depth)
    ├─ Miller columns (user list → folder list)
    │  ├─ Left: ALL users
    │  └─ Right: Selected user's folders only
    └─ Remove button (for selected folders)

  Tab 2: "Current Access List" (EXISTING)
    ├─ Item path (readonly)
    ├─ Permissions table
    └─ Remove Selected / Invite User buttons
    └─ AUTO-FETCHES when you select a folder (no fetch button)


COLOR CODING:

  Tab 2 (Current Access):
    • Green background   = owner
    • Blue background    = write permission
    • Yellow background  = read permission
    • Gray background    = inherited

  Tab 1 (Search Access):
    • Blue background    = write permission
    • Yellow background  = read permission
    • Gray background    = inherited (cannot remove)


SELECTION INDICATORS:

  Active Tab:
    • ▓Inverted background▓ shows which tab is currently active
    • Example: │▓Search Access  ▓│ Current Access List│

  Miller Column Selection:
    • Left column: ▓inverted background▓ shows selected user
    • Right column: ▓inverted background▓ shows selected folders (multi-select)
